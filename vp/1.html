vuePress-theme-reco 某君more    2020

入门篇

某君more 2020-09-20 vuepress

本篇文章主要介绍 vuepress 入门基础使用，学完本篇课程，你将会学会如何使用vuepress 搭建一个文档类型web页面

# 起步

前提条件

VuePress 需要 Node.js >= 8.6

  1. 创建并进入一个新目录vuepress-starter，windows下直接右键创建即可

    mkdir vuepress-starter && cd vuepress-starter

  2. 初始化项目

    npm init

  3. 将 VuePress 安装为本地依赖,官方已经不再推荐全局安装 VuePress

    npm install -D vuepress

  4. 创建你的第一篇文档

    • 在vuepress-starter目录下创建docs目录
    • 在docs目录下创建docs目录README.md文件
    • 在README.md文件 写入自己想要写的内容

  5. 在 package.json 中添加启动项目配置

    这个配置并非官方的，只是我自己习惯的vue项目

    {
  "scripts": {
    "serve": "vuepress dev docs",
    "build": "vuepress build docs"
  }
}

  6. 启动本地服务

    npm run serve

# 基本配置

如果没有任何配置，这个网站将会是非常局限的，用户也无法在你的网站上自由导航。为了更好地自定义你的网站，让我们首先在你的文档目录下创建一个 .vuepress 目录，所有 VuePress 相关的文件都将会被放在这里。你的项目结构可能是这样：

.
├─ docs
│  ├─ README.md
│  └─ .vuepress
│     └─ config.js
└─ package.json

一个 VuePress 网站必要的配置文件是 .vuepress/config.js，它应该导出一个 JavaScript 对象：

module.exports = {
  title: 'Hello VuePress', # 网站标题
  description: 'Just playing around' # 网站描述
}

对于上述的配置，如果你运行起 npm run serve，你应该能看到一个页面，它包含一个页头，里面包含一个标题和一个搜索框。VuePress 内置了基于 headers 的搜索 —— 它会自动为所有页面的标题、h2 和 h3 构建起一个简单的搜索索引。

# 构建首页

在docs目录下的README.md 中输入一下代码,这样编译就是一个默认主题提供的默认首页，你可以将相应的内容设置为 null 来禁用标题和副标题。

---
home: true
heroImage: /hero.png
heroText: Hero 标题
tagline: Hero 副标题
actionText: 快速上手 →
actionLink: /zh/guide/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。
- title: Vue驱动
  details: 享受 Vue + webpack 的开发体验，在 Markdown 中使用 Vue 组件，同时可以使用 Vue 来开发自定义主题。
- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。
footer: MIT Licensed | Copyright © 2018-present Evan You
---

# 导航栏

# 导航栏的基本使用

你可以在docs/.vuepress/config.js通过 themeConfig.nav 增加导航栏，代码如下

module.exports = {
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      { text: '指南', link: '/guide/' },
      { text: '外链', link: 'https://baidu.com' },
    ]
  }
}

# 导航栏下拉

当你提供了一个 items 数组而不是一个单一的 link 时，它将显示为一个 下拉列表 ：

module.exports = {
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      { text: '指南', link: '/guide/' },
      { text: '外链',
       items: [
          { text: '百度', link: 'https://www.baidu.com/' },
          { text: 'vuepress', link: 'https://www.vuepress.cn/' }
        ]
      }
    ]
  }
}

# 导航栏下拉分组

此外，你还可以通过嵌套的 items 来在 下拉列表 中设置分组：

module.exports = {
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      {
        text: '指南',
        items: [
          {
            text: 'api',
            items: [
              { text: 'cli', link: '/api/cli' },
              { text: 'node', link: '/api/node' }
            ]
          },
          {
            text: '开发指南',
            items: [
              { text: '本地开发', link: '/develop/host' },
              { text: '设计理念', link: '/develop/design' }
            ]
          }
        ]
      },
      {
        text: '外链',
        items: [
          { text: '百度', link: 'https://www.baidu.com/' },
          { text: 'vuepress', link: 'https://www.vuepress.cn/' }
        ]
      }
    ]
  }
}

# 侧边栏

# 侧边栏的基本使用

想要使 侧边栏（Sidebar）生效，需要配置 themeConfig.sidebar，基本的配置，需要一个包含了多个链接的数组：

module.exports = {
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      {
        text: '指南',
        items: [
          {
            text: 'api',
            items: [
              { text: 'cli', link: '/api/cli' },
              { text: 'node', link: '/api/node' }
            ]
          },
          {
            text: '开发指南',
            items: [
              { text: '本地开发', link: '/develop/host' },
              { text: '设计理念', link: '/develop/design' }
            ]
          }
        ]
      },
      {
        text: '外链',
        items: [
          { text: '百度', link: 'https://www.baidu.com/' },
          { text: 'vuepress', link: 'https://www.vuepress.cn/' }
        ]
      }
    ],
    sidebar: [
      '/develop/',
      '/develop/design',
      ['/develop/host','开发指南']
    ]
  }
}

提示

你可以省略 .md 拓展名，同时以 / 结尾的路径将会被视为 */README.md，这个链接的文字将会被自动获取到（无论你是声明为页面的第一个 header，还是明确地在 YAML front matter 中指定页面的标题）。如果你想要显示地指定链接的文字，使用一个格式为 [link, text] 的数组

# 侧边栏分组

你可以通过使用对象来将侧边栏划分成多个组：

module.exports = {
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      {
        text: '指南',
        items: [
          {
            text: 'api',
            items: [
              { text: 'cli', link: '/api/cli' },
              { text: 'node', link: '/api/node' }
            ]
          },
          {
            text: '开发指南',
            items: [
              { text: '本地开发', link: '/develop/host' },
              { text: '设计理念', link: '/develop/design' }
            ]
          }
        ]
      },
      {
        text: '外链',
        items: [
          { text: '百度', link: 'https://www.baidu.com/' },
          { text: 'vuepress', link: 'https://www.vuepress.cn/' }
        ]
      }
    ],
    sidebar: [{
      title: '基础篇',   // 必要的
      // path: '/develop/',      // 可选的, 标题的跳转链接，应为绝对路径且必须存在
      collapsable: false, // 可选的, false 让一个组永远都是展开状态，默认值是 true,
      sidebarDepth: 3,    // 可选的, 默认值是 1 获取到h2.等3时获取到h4
      children: [
        '/develop/',
        '/develop/design'
      ]
    },
    {
      title: '高级篇',   // 必要的
      // path: '/develop/',      // 可选的, 标题的跳转链接，应为绝对路径且必须存在
      collapsable: false, // 可选的, false 让一个组永远都是展开状态，默认值是 true,
      sidebarDepth: 3,    // 可选的, 默认值是 1 获取到h2.等3时获取到h4
      children: [
        ['/develop/host', '开发指南']
      ]
    }
    ]
  }
}

# 多侧边栏分组

如果你想为不同的页面组来显示不同的侧边栏，首先，将你的页面文件组织成下述的目录结构：

.
├─ README.md
├─ contact.md
├─ about.md
├─ foo/
│  ├─ README.md
│  ├─ one.md
│  └─ two.md
└─ bar/
   ├─ README.md
   ├─ three.md
   └─ four.md

注意

确保 fallback 侧边栏被最后定义。VuePress 会按顺序遍历侧边栏配置来寻找匹配的配置

接着，遵循以下的侧边栏配置：

module.exports = {
  themeConfig: {
    sidebar: 'auto',
    nav: [
      { text: '首页', link: '/' },
      {
        text: '指南',
        items: [
          {
            text: 'api',
            items: [
              { text: 'cli', link: '/api/cli' },
              { text: 'node', link: '/api/node' }
            ]
          },
          {
            text: '开发指南',
            items: [
              { text: '本地开发', link: '/develop/host' },
              { text: '设计理念', link: '/develop/design' }
            ]
          }
        ]
      },
      {
        text: '外链',
        items: [
          { text: '百度', link: 'https://www.baidu.com/' },
          { text: 'vuepress', link: 'https://www.vuepress.cn/' }
        ]
      }
    ],
    sidebar: {
      '/foo/': [
        '',     /* /foo/ */
        'one',  /* /foo/one.html */
        'two'   /* /foo/two.html */
      ],

      '/bar/': [
        {
          title: '初级',  
          children: [
            '',     
            'three', 
            'four' 
          ]
        },
        {
          title: '高级',  
          children: [
            '',     
            'three', 
            'four' 
          ]
        }
      ],

      // fallback
      '/': [
        '',        /* / */
        'contact', /* /contact.html */
        'about'    /* /about.html */
      ]
    }
  }
}

# 配置抽离（可选）

通过以上配置后，你会发现一个问题，随着配置的增多，config.js文件将会变得越来越庞大，代码越来越复杂，这样对我们后期维护与修改很不利，所以这时候需要对config.js进行代码抽离的配置

# 抽离导航栏

  1. 新建一个nav.js文件，将config.js中nav对象拷贝进来，代码如下

    module.exports = [
    { text: '首页', link: '/' },
    {
        text: '指南',
        items: [
            {
                text: 'api',
                items: [
                    { text: 'cli', link: '/api/cli' },
                    { text: 'node', link: '/api/node' }
                ]
            },
            {
                text: '开发指南',
                items: [
                    { text: '本地开发', link: '/develop/host' },
                    { text: '设计理念', link: '/develop/design' }
                ]
            }
        ]
    },
    {
        text: '外链',
        items: [
            { text: '百度', link: 'https://www.baidu.com/' },
            { text: 'vuepress', link: 'https://www.vuepress.cn/' }
        ]
    }
]

  2. config.js 代码调整成如下代码

    module.exports = {
  themeConfig: {
    sidebar: 'auto',
    nav: require("./nav.js"),
    sidebar: {
      '/foo/': [
        '',     /* /foo/ */
        'one',  /* /foo/one.html */
        'two'   /* /foo/two.html */
      ],

      '/bar/': [
        {
          title: '初级',  
          children: [
            '',     
            'three', 
            'four' 
          ]
        },
        {
          title: '高级',  
          children: [
            '',     
            'three', 
            'four' 
          ]
        }
      ],

      // fallback
      '/': [
        '',        /* / */
        'contact', /* /contact.html */
        'about'    /* /about.html */
      ]
    }
  }
}
