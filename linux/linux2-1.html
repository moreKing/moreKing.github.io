# 1.为什么使用LVM

LVM（Logical Volume Manager）逻辑卷管理，是在硬盘分区和文件系统之间添加的一个逻辑层，为文件系统屏蔽下层硬盘分区布局，并提供一个抽象的盘卷，在盘卷上建立文件系统。

比如我们有一个硬盘/dev/sda。分了3个主分区:sda1,sda2,sda3,后面磁盘空间没有划分，分别对应的挂载点是/boot,/,/home,如果我们的/home空间不够了,怎么办?

  1. 传统的方法是在未划分的空间中分割一个分区，挂载到/home下，并且把sda3的内容复制到这个新分区上.
  2. 或者把这个新分区挂载到另外的挂载点上,然后在/home下创建链接,链接到这个新挂载点。

这两种方法都不大好,第一种方法浪费了sda3,并且如果后面的分区容量小于sda3怎么办?第二种方法不大浪费容量,但是每次要额外创建链接,比较麻烦.那么,如果用lvm呢?lvm的好处在于,可以动态放大一个逻辑卷(相当于一个逻辑分区),也就是说,sda3如果是一个逻辑分区,比如/dev/rootvg/lv3,那么lv3可以被动态放大.这样就解决了动态容量调整的问题。

# 2.基本介绍

# 2.1 基础术语

  • 物理存储介质（The physical media）：LVM存储介质，可以是硬盘分区、整个硬盘、raid阵列或SAN硬盘。设备必须初始化为LVM物理卷，才能与LVM结合使用。
  • 物理卷PV（physical volume）：物理卷就是LVM的基本存储逻辑块，但和基本的物理存储介质比较却包含与LVM相关的管理参数，创建物理卷可以用硬盘分区，也可以用硬盘本身。
  • 卷组VG（Volume Group）：LVM卷组类似于非LVM系统中的物理硬盘，一个卷组VG由一个或多个物理卷PV组成。可以在卷组VG上建立逻辑卷LV。
  • 逻辑卷LV（logical volume）：类似于非LVM系统中的硬盘分区，逻辑卷LV建立在卷组VG之上。在逻辑卷LV之上建立文件系统。
  • 物理块PE（physical Extent）：物理卷PV中可以分配的最小存储单元，PE的大小可以指定，默认为4MB
  • 逻辑块LE（Logical Extent）：逻辑卷LV中可以分配的最小存储单元，在同一卷组VG中LE的大小和PE是相同的，并且一一相对。

由上图可以得出LVM的大致思路

  1. 将物理磁盘或者磁盘分区创建成物理卷（PV），并且进行初始化PV
  2. 将多个PV组合成VG卷组，相当于把多个磁盘或分区组合成一个大的“磁盘”
  3. 从VG中划分出逻辑卷（LV），相当于把一个大的磁盘（VG）进行了分区，可以创建多个LV
  4. 格式化LV，挂载到对应目录下进行使用
  5. 设置开机自动挂载

# 2.2 LVM的优点

由上面的 思路我们可以得知LVM无论有多少个分区或者物理硬盘，都可组成一个大的VG（卷组）

  • 空间可以充分的利用
  • LV空间不足可以动态调整，不用担心存储的连续性，最重要的是不影响业务
  • VG、LV可以在线动态的进行创建、删除、调整大小等操作
  • LVM允许创建快照，用来保存文件系统的备份。（正常情况用不到）

# 2.3 LVM常用命令

功能 PV管理命令 VG管理命令 LV管理命令
scan 扫描 pvscan vgscan lvscan
create 创建 pvcreate vgcreate lvcreate
display 显示 pvdisplay vgdisplay lvdisplay
remove 移除 pvremove vgremove lvremove
extend 扩展 vgextend lvextend
reduce 减少 vgreduce lvreduce

# 4. 挂载后重启问题

当我们挂载完成后，重启服务器，有可能会遇到以下问题，这个问题表面再设置开启挂载硬盘时候，配置有问题，解决方法需要进入单用户模式进入系统修改这个文件

A start job is running for dev-**********

  1. 开机进入如下界面时。按 **E **

  2. 按方向键下，定位到最后，找到ro一行，ro的意思是read only，将ro替换成 rw init=/sysroot/bin/sh，如下图

  3. 按Ctrl-x 进行重启进入单用户模式

  4. 执行chroot /sysroot。其中chroot命令用来切换系统，/sysroot/目录就是原始系统，然后再次编辑 /etc/fstab

  5. 按下Ctrl+D后，执行reboot

此时这个问题就解决了！
